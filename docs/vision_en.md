# 项目重构目标

说明：这个文件保留了原来的英文文件名，但内容已经统一改成中文。更完整的中文目标文档见 [vision_zh.md](</C:/Users/mingy/Documents/New project/CCompiler/docs/vision_zh.md>)。

## 1. 背景

这是一个已有的 C++ 编译器 / 代码生成项目。项目中有较多历史包袱：

- 自定义基础类型较多。
- 手写资源管理较多。
- 对 Windows 和 x86 后端存在较强耦合。
- 命名风格不统一。
- 文档不足。
- 部分历史注释可能存在旧编码问题。

本次重构目标不是推倒重写，而是在原项目中渐进式现代化。

## 2. 核心目标

### 2.1 原地渐进式重构

重构应保留现有仓库和工程结构，优先小批次、可验证推进。

### 2.2 用标准 C++ 替换历史基础设施

重点替换或压缩以下自定义设施的使用面：

- 自定义字符串。
- `List`、`LinkedList`、字典类结构。
- `RefPtr` 等自定义智能指针。
- 自定义文件流和文本流包装。

优先使用：

- `std::string` / `std::wstring`
- `std::vector`
- `std::unordered_map` / `std::map`
- 标准智能指针
- `<fstream>` 和标准 I/O 设施

公共接口不必一次性全部替换，可以先替换内部实现和局部所有权。

### 2.3 减少 Windows 专有依赖

当前不要求立即跨平台，但应避免继续引入新的 Windows 专属基础设施，并在可行处使用标准 C++。

## 3. 质量目标

- 可读性提升。
- 命名更统一。
- 修复真实 correctness 问题。
- 保持主构建可用。
- 每批改动后验证。

## 4. 文档目标

- 补充模块说明。
- 说明保留兼容层的原因。
- 恢复或重写旧注释中的有效知识。
- 尽量统一到 UTF-8 和清晰中文说明。

## 5. 阶段顺序

1. 稳住并现代化内部实现。
2. 缩减自定义容器和手工资源管理使用面。
3. 增加最小回归测试。
4. 压缩兼容层暴露面。
5. 最后再考虑大范围公共接口替换。

## 6. 当前阶段不优先做

- LLVM 风格 IR 整体重做。
- CFG 可视化工具重写。
- 完整跨平台化。
- 仓库结构大重组。
- x86 后端新功能扩展，除非阻塞当前主线验证。

## 7. 工作原则

当目标冲突时：

- 正确性优先于表面整理。
- 渐进可验证优先于大而不稳。
- 内部现代化优先于公共接口大破坏。
- 清晰优先于炫技。
- 标准库优先于重复造轮子。
